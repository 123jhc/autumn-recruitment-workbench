# v1.1.0 版本说明

日期：2026-07-13

## 版本概述

在 v1.0.0 基础上新增 AI 多配置管理、仪表盘布局优化、以及边界场景修复。

## 新增功能

### AI 配置管理优化

支持网页端管理多个 AI 配置，单选激活、连通性测试，无需重启服务。

**服务端：**
- 配置存储：`server/src/services/ai/config-store.ts`，读写 `server/ai-configs.json`，Promise 链串行化保证原子落盘
- 首次引导：`.env` 三项（`AI_BASE_URL`/`AI_API_KEY`/`AI_MODEL`）齐全时自动生成"默认"配置并激活；之后以 JSON 文件为准
- 路由：`server/src/routes/ai-configs.ts`，提供 CRUD + 激活 + 连通测试 6 个端点
- 连通测试：`callAiProbe` 发 1-token 轻量请求验证真实连通性，返回 `{ok, message}` 而不抛异常
- 安全：`apiKey` 仅存服务端 JSON 文件，前端视图 `AiConfigView` 仅含 `hasApiKey` 布尔

**前端：**
- `AiConfigList`：配置列表，单选激活（◉ / ○），每行测试/编辑/删除
- `AiConfigForm`：Drawer 内表单，编辑时回显当前值、Key 留空不修改
- `AiStatus`：顶部概览，显示激活配置名称/模型 + 连通徽章（可用 ✓ / 不可用 / 未测试）
- `SettingsContext` 支持多配置状态：`aiConfigs` / `activeId` + `loadConfigs` / `createConfig` / `updateConfig` / `deleteConfig` / `setActive` / `testConfig`

**契约：**
- 编辑配置时 apiKey 留空 → 服务端保留原 key，不修改
- 连通测试失败 → 返回 200 + `{ok:false, message}`，前端 toast 展示可操作消息
- 删除激活配置 → `activeId` 置 null，AI 回到"未配置"状态

### 仪表盘布局优化

- `DashboardPage` 纵向卡片布局，各模组统一间距与顺序
- 压缩空状态文案，辅助区域使用简洁提示语
- 新增 `DashboardPage.test.tsx` 验证布局结构和唯一新建入口

## Bug 修复

- **岗位投递编辑不回显旧内容**：`ApplicationForm` 常驻挂载导致 `useState` 不重置，加 `key={editingApp?.id ?? 'new'}` 强制重挂载
- **无 body 请求带 Content-Type 报错**：`api-client.ts` 的 `request` 辅助函数仅在 `body` 非空时设置 `Content-Type: application/json`
