# v1.0.0 版本说明

日期：2026-07-13

## 版本概述

秋招准备工作台首版，完整实现了设计规格中定义的全部功能。

## 技术栈

| 层 | 技术 |
|----|------|
| 前端 | Vite 6 + React 19 + TypeScript |
| 路由 | React Router 7 |
| 样式 | CSS Modules |
| 本地数据 | Dexie.js 4 (IndexedDB) |
| 服务端 | Fastify 5 + TypeScript |
| AI | OpenAI 兼容接口（本地代理） |
| 测试 | Vitest 3 |

## 项目结构

```
  shared/          # 共享类型、Zod schema、常量（3 个子模块）
  client/          # 前端（Vite + React）
    dal/           # 数据访问层（6 个模块：task/task-draft/plan-import/application/leetcode/backup）
    contexts/      # React Context（Task/Application/LeetCode/Settings）
    pages/         # 5 个页面（Dashboard/Tasks/Applications/LeetCode/Settings）
    components/    # 10 个共享 UI 组件
    hooks/         # 日期工具（Asia/Shanghai 时区）
    services/      # API 客户端
    db/            # Dexie 数据库定义
  server/          # 本地服务端（Fastify）
    routes/        # /api/plan/extract, /api/plan/parse, /api/ai/status
    services/      # 文件提取（MD/DOCX/PDF）+ AI 代理
```

## 已实现功能

### 今日总览
- 今日和逾期任务展示，支持直接完成/取消完成
- 本周完成进度条
- 岗位下一步行动（7 天内到期）
- LeetCode 复习提醒

### 计划与任务
- 手动新建/编辑/删除任务（不依赖 AI）
- 文件上传（.md/.txt/.docx/.pdf）和粘贴文本
- AI 拆解为任务草稿，草稿可编辑/删除/补充
- 确认后单个 IndexedDB 事务批量写入正式任务
- 按视图（今天/本周/全部/已完成）和分类筛选

### 岗位投递
- CRUD 岗位信息（公司/岗位/地点/链接/要求/状态/日期/备注）
- 7 种状态：目标岗位→准备投递→已投递→笔试→面试→Offer→拒绝
- 状态数量统计面板，搜索和状态筛选
- 下一步行动一键生成关联任务（含去重检查）
- 岗位删除后保留关联任务，显示"来源已删除"

### LeetCode
- CRUD 题目信息（题号/标题/链接/难度/标签/状态/解法/日期）
- 3 种状态：待刷/已完成/需复习
- 按难度、标签、状态筛选
- 本周完成统计（数量+难度分布）
- 到期复习提醒展示在今日总览，支持完成复习操作

### 设置与备份
- AI 服务配置状态显示（已配置/未配置）
- JSON 完整备份导出（含 schemaVersion）
- 备份导入前 Zod 校验+摘要展示，确认后单事务整库替换
- 恢复失败自动回滚，API Key 不包含在备份中

## 关键设计

- AI 只生成草稿，用户确认后才写入（事务保证原子性）
- API Key 仅存服务端 .env，永不泄露到前端或备份
- 所有非 AI 功能在无 AI 配置下正常工作
- 日期按 Asia/Shanghai 时区，周一至周日为本周
