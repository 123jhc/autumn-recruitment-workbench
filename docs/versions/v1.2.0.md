# v1.2.0 版本说明

日期：2026-07-14

## 版本概述

新增 **LeetCode 热题 100 刷题规划**：离线内置 100 道官方热题，按 17 个专题连续刷题；用户选择开始日期、截止日期和具体星期，预览后生成每日计划。同步完成 LeetCode 数据模型规范化、Dexie v2 迁移、备份 schema v2，以及 shared workspace 构建修复。

> 本版本对应交接记录：`docs/superpowers/handoffs/2026-07-14-leetcode-hot-100.md`

## 新增功能

### LeetCode 热题 100 规划

**静态题单：**
- 新增 `client/src/features/leetcode/hot100.ts`，内置 100 道题的题号、slug、中文标题、难度、标签、主专题、官方题单顺序和链接
- 17 个专题顺序：哈希、双指针、滑动窗口、子串、普通数组、矩阵、链表、二叉树、图论、回溯、二分查找、栈、堆、贪心、动态规划、多维动态规划、技巧
- 离线内置，不运行时抓取题单、不保存正文/题解、不同步账号

**排期与队列纯函数**（`client/src/features/leetcode/planning.ts`）：
- `buildSchedule`：日期首尾包含、星期 1–7、未完成题均匀分配、余数落较早日期、排除已完成题、明确校验错误
- `moveInQueue`：只调整未完成队列，已完成题顺序不变
- `filterProblemRows`：难度/状态/专题/标签 AND 语义组合筛选

**初始化 / 重新排期弹窗**（`PlanSetupModal`）：
- 选择开始日期、截止日期、每周刷题日
- 预览：新增/匹配题数、刷题日数、每日题量区间、各专题题量
- 初始化时检测题号/slug 冲突，slug 优先匹配、题号回退匹配，冲突时阻断
- 「重新排期」仅重排未完成题，已完成题保持原 `plannedDate`

**LeetCode 页面**（`LeetCodePage`）：
- 空状态初始化入口
- 今日计划 / 全部题目 / 按专题三种视图
- 完成题目、前移、后移、重新排期入口
- Context 始终加载完整数据；筛选只生成 `filteredProblems`，周统计与复习提醒基于完整数据集，不受筛选污染

**今日总览**：仪表盘新增 `TodayLeetCode` 卡片，展示当天题目，可直接打开题目和完成

## 数据模型规范化与迁移

### 类型与 Schema（`shared/`）

LeetCode 类型拆出（`shared/src/types/leetcode.ts`）：

| 类型 | 用途 |
|---|---|
| `LeetCodeCatalogProblem` | 题目目录（slug、题号、标题、链接、难度、标签、来源） |
| `LeetCodeListEntry` | 题单关系（题单 ID、slug、专题、推荐顺序） |
| `LeetCodeProgress` | 个人进度（slug、状态、计划/完成日期、队列顺序、解题摘要） |
| `LeetCodeReviewRecord` | 复习记录（待复习/已完成） |
| `LeetCodeSchedule` | 排期配置（题单 ID、起止日期、每周刷题日） |
| `LeetCodeProblem` | 页面用联表读模型，**不作为 Dexie 单表写入** |
| `LegacyLeetCodeProblem` | 仅保留用于 v1 迁移 |

对应 Zod schema 已补齐（`shared/src/schemas/leetcode.schema.ts`）；`LeetCodeProblemSchema` 作为 v1 导入兼容别名。

### Dexie 升级至 v2（`client/src/db/database.ts`）

- 新增五张规范化表：`leetCodeCatalog` / `leetCodeListEntries` / `leetCodeProgress` / `leetCodeReviews` / `leetCodeSchedules`
- 升级回调把旧 `leetCodeProblems` 迁入目录/进度/复习记录，迁完清空旧表
- 迁移逻辑见 `client/src/features/leetcode/legacy-normalization.ts`：按 slug 分组，状态最优候选保留进度与复习记录

### DAL（`client/src/dal/leetcode.dal.ts`）

规范化存储，提供：
- 初始化热题 100（slug 优先、题号回退匹配，冲突阻断，旧进度保留）
- 重新排期、队列移动、完成题目、复习完成、手工题 CRUD
- 联表查询供页面使用
- 复习状态由「进度已完成 + 存在未完成复习记录」派生为 `review`

### 备份 schema v2（`shared/src/schemas/backup.schema.ts`）

- `BackupEnvelope` 改为 `schemaVersion` 1/2 判别联合
- v2 导出五类 LeetCode 规范化数据
- 导入继续接受 v1 并迁移（`backup.dal.ts`），设置页备份摘要兼容 v1/v2

## Bug 修复 / 工程修复

- **shared workspace 缺失 build 脚本**：`shared/package.json` 补 `"build": "tsc"`，根 `npm run build` 的 `shared -> client -> server` 顺序链生效（v1.1.0 备注「shared 无 build 脚本，需事先存在」至此解决）
- LeetCode 调度边界保护：开始日期晚于截止日期、所选范围无刷题日等明确报错
- LeetCode DAL 不变量保留：事务写入、冲突阻断、只重排未完成题

## 范围边界

- 不同步账号、不运行时抓取题单、不保存正文/题解、不做自动复习间隔
- LeetCode 每日题不复制进通用 `Task`，今日总览仅派生展示
- 题号/slug 与标题等元数据建议在最终手动验收前抽查
